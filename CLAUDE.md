## Environment Setup

Copy `.env.example` to `.env`. Start PostgreSQL with:

```bash
docker compose up -d
```

The database runs on port 5432 (kafe/postgres/postgres).

## Architecture

This is a NestJS API following clean architecture with strict layer separation:

```
src/
├── domain/                    # Core business rules — no framework dependencies
│   ├── entities/              # Immutable value objects (User, Product, Order)
│   ├── repositories/          # Abstract interfaces (IUserRepository, etc.)
│   └── errors/                # Domain exceptions (ConflictError, etc.)
├── application/use-cases/     # Business logic orchestration
│   ├── users/
│   ├── menu/
│   ├── orders/
│   └── inventory/
├── infrastructure/            # Concrete implementations
│   ├── db/
│   │   ├── schema.ts          # Drizzle ORM table definitions
│   │   ├── auth-schema.ts     # Better-Auth managed tables
│   │   ├── repositories/      # Drizzle implementations of domain interfaces
│   │   └── migrations/        # Auto-generated by drizzle-kit
│   └── auth/                  # Better-Auth config and NestJS module
├── presentation/              # HTTP layer
│   ├── controllers/           # Route handlers
│   ├── dtos/                  # Request/response validation
│   ├── decorators/            # @CurrentUser() etc.
│   ├── filters/               # HttpExceptionFilter → standardized error responses
│   └── interceptors/          # TransformInterceptor → standardized success responses
└── test/repositories/         # In-memory repository implementations for unit tests
```

Feature modules (`users.module.ts`, `menu.module.ts`, `orders.module.ts`) wire everything together via NestJS DI: controllers → use cases → repository interface → Drizzle implementation.

## Key Patterns

**Repository pattern**: Domain defines interfaces (`IUserRepository`), infrastructure provides Drizzle implementations, test layer provides in-memory fakes. Use cases only depend on the interface.

**Use case injection**: Each use case is registered in the module with a factory:
```typescript
{ provide: CreateUserUseCase, useFactory: (repo) => new CreateUserUseCase(repo), inject: [IUserRepository] }
```

**Unit testing**: Instantiate use cases with in-memory repositories from `src/test/repositories/`. No database or NestJS bootstrapping needed.

**Authentication**: Better-Auth with Bearer token plugin. Protected routes use `@Roles(['ADMIN'])` from `@thallesp/nestjs-better-auth`. Extract current user with `@CurrentUser()` decorator.

**Response format**: `TransformInterceptor` wraps all success responses; `HttpExceptionFilter` converts domain errors to HTTP responses. Domain errors (`ConflictError`, etc.) carry their own HTTP status codes.

## API

- Base path: `/api/v1`
- Swagger docs: `/api/v1/docs`
- Auth endpoint: `POST /api/v1/auth/login` → returns `{ token, user }`

## Database Schema

ORM: Drizzle with PostgreSQL. Key tables: `categories`, `products`, `ingredients`, `product_ingredients`, `orders`, `order_items`, `inventory_movements`. Enums: `user_role` (ADMIN/BARISTA/CLIENT), `order_status` (RECEIVED/IN_PREPARATION/READY/DELIVERED/CANCELLED), `movement_type` (DEDUCTION/RESTOCK/ADJUSTMENT).

Schema is split across `schema.ts` (business tables) and `auth-schema.ts` (Better-Auth managed: users, sessions, accounts).
